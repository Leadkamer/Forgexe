# B2B Homepage Blueprint

A research-backed, conversion-optimized blueprint for building B2B service/agency homepages. Based on convergent findings from NNGroup, Unbounce, HubSpot, StoryBrand, CXL, and practical implementation.

## When to use this skill

- Building a new homepage for a B2B service business
- Restructuring an existing homepage for better conversion
- Reviewing homepage copy and section order
- Advising on CTA placement, social proof, and trust signals
- Any homepage project targeting SMB/MKB clients

## The 10-Section Blueprint

Every section has a specific psychological function. The order is non-negotiable — it maps to documented cognitive patterns (primacy effect, progressive disclosure, commitment-consistency).

### Section 1: Hero (Above the Fold)

**Function:** Pass the "grunt test" — within 3 seconds answer: What do you do? How does it make my life better? What do I do next?

**Requirements:**
- Benefit-driven headline, 10-15 words max
- Subheadline clarifying who you serve
- ONE primary CTA button (multiple CTAs decrease conversions up to 266%)
- Supporting visual (product screenshot, illustration, or animation — never generic stock)
- Risk reducers below CTA ("Free, no obligations, response within X hours")

**Key data:**
- 57% of page viewing time is spent above the fold (NNGroup, 130K eye fixations)
- Pages at 5th-7th grade reading level convert at 11.1% vs 5.3% for college-level (Unbounce)
- Personalized CTAs convert 202% better than generic ones (HubSpot, 330K CTAs)

**Copy principles:**
- Lead with outcome, not capability ("More customers, less manual work" > "AI-powered automation platform")
- Use the client's language, not industry jargon
- Frame the business as the guide, not the hero (StoryBrand)

**Common mistakes:**
- Full-screen hero images that create "illusion of completeness" — 75% of users failed to scroll past (NNGroup)
- Fix: ensure content visually "peeks" above the fold line

### Section 2: Social Proof Bar

**Function:** Leverage bandwagon effect at the exact moment the visitor decides whether to invest more cognitive effort.

**Requirements:**
- Place immediately below hero or within it
- 4-10 grayscale client logos with framing line ("Trusted by X+ businesses")
- Logos don't need to be Fortune 500 — relatable mid-market brands work better for SMB targeting

**Key data:**
- 79% of B2B buyers rely on social proof when making decisions (Intelemark)

**Implementation notes:**
- Grayscale logos with hover-to-color effect
- For SMB agencies: local/recognizable brands > big names
- If no logos yet: use testimonial snippets or client count

### Section 3: Problem Statement (Challenge-First)

**Function:** Name the visitor's pain in their own language before presenting solutions. Creates emotional tension that makes Section 4 feel like relief.

**Two approaches:**

**A) Static problem cards (simpler):**
- 3 specific pain points the target client faces
- Concrete frustrations, not abstract challenges
- Example: "You lose leads because nobody responds fast enough"

**B) Interactive challenge selector (higher engagement, recommended):**
- "Which challenge are you facing?" — 4-6 clickable cards
- Each click reveals the relevant solution
- Each click is a micro-commitment (Cialdini's commitment-consistency)
- Inspired by Hiper.ai pattern

**Key data:**
- PAS (Problem-Agitation-Solution) framework tested on SweatBlock.com: 108% lift in revenue (Copyhackers)

### Section 4: Solution Process (3 Steps)

**Function:** Reduce cognitive load. Hick's Law proves decision time increases logarithmically with choices. Three steps make engagement feel achievable.

**Requirements:**
- Exactly 3 steps (StoryBrand "The Plan")
- Each step: short title + 1-2 sentence description
- Step 1 should match primary CTA (e.g., "Book free scan")
- Include CTA button after steps reflecting Step 1

**Pattern:**
1. Diagnose / Scan / Audit (free, low barrier)
2. Build / Implement / Deploy (you do the work)
3. Results / Growth / Runs autonomously (outcome)

### Section 5: Value Propositions / Service Pillars

**Function:** Present 3-6 core value propositions. Miller's Law: people hold 5-9 items in working memory — never exceed this.

**Requirements:**
- Frame as outcomes, not features
- Icons or illustrations per item
- Each item no longer than 2-3 sentences

**For agencies with multiple service lines:**
- Group under 2-3 pillars (e.g., "Growth" + "Operations")
- Sub-services listed under each pillar
- Overarching services (training, consulting) as separate wide blocks below pillars

**Visual approach:**
- Line illustrations in brand style (Cumlaude.ai reference)
- Each expertise card: illustration + title + 1-line description

### Section 6: Quantified Results & Case Studies

**Function:** Deploy strongest proof. Bold headline metrics are instantly scannable during F-pattern sweep.

**Requirements:**
- 3-4 bold KPI counters with animated counting
- 2-4 case metric cards with specific outcomes
- Link each to full case study page

**Key data:**
- 73% of B2B buyers use case studies when making purchasing decisions (Heinz Marketing)

**Format:**
- Counter: "147%" + label "growth in qualified leads"
- Case card: Type tag (Growth/Ops) + Company description + Result line
- Use real numbers — specificity converts

### Section 7: Testimonials

**Function:** Named quotes with transformation narratives, placed strategically before a CTA.

**Requirements:**
- 2-3 testimonials with real names, titles, companies
- Star ratings if available
- Bold headline per testimonial summarizing the transformation
- CTA button IMMEDIATELY after testimonials

**Key data:**
- CTA placed after testimonials increases conversions by 25% (Amra & Elma)

**What makes a good testimonial:**
- Mirrors problem-solution narrative
- Includes specific outcomes ("15 hours/week saved", "ROI in 2 months")
- Not generic praise ("Great agency!")

### Section 8: Authority Signals + FAQ

**Tech stack / tools:**
- Show tools/platforms you work with
- Visual approach: 3D layered visualization, periodic table monograms, or logo grid
- Grayscale with hover-to-color

**FAQ (3-4 questions):**
- Address top objections: pricing, timeline, migration, differentiation
- Expandable accordion format
- Keep answers concise (2-3 sentences)

**Key data:**
- B2B websites with key certifications achieve up to 40% higher conversions (ProVisors)

### Section 9: Pricing

**Function:** Transparency reduces friction. SMB buyers won't spend time on a call just to learn price.

**Requirements:**
- 3 tiers (Good/Better/Best)
- Highlight middle tier as "Most chosen"
- Starting prices or price ranges — never hide completely
- Each tier: name, description, price, feature list, CTA
- If custom pricing: "Get your quote in 24 hours"

**Implementation:**
- Use ">" instead of checkmarks for feature lists (terminal aesthetic for tech brands)
- Price amount in brand accent color, large font
- CTA per card linking to contact

### Section 10: Final CTA (Risk-Reducing)

**Function:** Serial position effect's recency bias — second-most-remembered section after hero.

**Requirements:**
- Strong closing headline
- Repeat primary CTA
- 3 risk reducers ("Free", "Report within 48 hours", "Response within 2 hours")
- 2-3 contact options (phone, email, WhatsApp/chat)
- Lower-commitment secondary CTA for visitors not ready to buy

**Key data:**
- CTA at bottom of long-form page increased conversions by 304% (Michael Aagaard)

### Footer

- Contact info, simplified nav, social links
- 4-column grid: Brand, Company links, Services/Branches, Contact
- 30% of users check footer for security/credibility (research)
- For SMB: include local phone number and physical address

## Cross-Cutting Principles

### CTA Strategy
- ONE primary CTA repeated 3-4 times throughout page
- Sticky CTA in navigation (always visible)
- Lower-barrier language: "Get your free growth plan" > "Request a proposal"
- Sticky CTAs increase mobile conversions 15-40%

### Social Proof Distribution
- Don't cluster all proof in one block
- Layer throughout: logos near hero, metrics in middle, testimonials after process, badges near CTAs
- Every scroll position should encounter at least one proof element

### Typography & Readability
- Write at 5th-7th grade level (Unbounce: 62% stronger correlation between difficult vocabulary and declining conversions in 2024 vs 2020)
- Minimum 16px body text for mobile
- Scale up ~20-30% from typical web sizes for premium feel
- Use serif for headlines (authority) + monospace for UI elements (technical credibility)

### Mobile-First
- 82.9% of landing page visitors arrive on mobile (Unbounce)
- Sticky CTA bar on mobile
- Single-column layouts for complex sections
- Touch-friendly card sizes (min 44px tap targets)

### Reading Patterns
- Z-pattern for visual/hero content — primary CTA at bottom-right of Z
- F-pattern for text-heavy sections — front-load key info on left
- Avoid horizontal rules creating "false floors" that suppress scrolling

### Speed
- 53% of mobile visitors abandon pages >3 seconds load (Google)
- Inline SVG illustrations > external images
- Base64 embed small logos
- Minimize external font weights

## Common Mistakes That Kill Conversions

1. **Vague value proposition** — "Crafting transformative digital experiences" tells nothing
2. **Slow page load** — reducing 6s to 2s = 40% more form submissions
3. **Navigation overload** — use 5-7 main menu items max
4. **Full-screen hero** suppressing scroll (illusion of completeness)
5. **Burying results** below 3rd screenful (only 19% viewing time there)
6. **Capability brochure** instead of conversion tool
7. **Not optimizing for mobile** despite 82.9% mobile traffic
8. **Multiple competing CTAs** — decreases conversions up to 266%

## Implementation Notes for Artifact/HTML Builds

### Dark/Light Mode
- Use CSS custom properties on body element
- Toggle via body class (body.light), not data-theme (artifact viewers block that)
- addEventListener instead of inline onclick
- No localStorage in artifact viewer — use class toggle only

### SVG Illustrations
- Use var(--css-variable) in fill/stroke for theme-adaptive illustrations
- Keep SVG paths simple to avoid Cloudflare email protection corruption
- Use monospace font-family="monospace" in SVG text (not var(--font))
- Inline SVGs for reliability, no external dependencies

### Email Addresses
- Write as "name at domain.nl" to prevent Cloudflare email protection injection
- Replace with real mailto: links only on production hosting

### Logo Embedding
- Base64 encode client logos directly in HTML for artifact viewer compatibility
- Remove white/black backgrounds programmatically (PIL: pixels > 235 or < 30 → transparent)
- Apply grayscale + opacity filter, color on hover
- Special CSS classes for originally-white logos (.logo-white) with brightness/contrast adjustments

### Animation Patterns That Work in Artifacts
- CSS @keyframes for floating, pulsing, orbit effects
- IntersectionObserver for scroll-triggered reveals
- Counter animations via requestAnimationFrame
- Staggered animation-delay for sequential entrance effects
- Traditional function() syntax over arrow functions for compatibility
